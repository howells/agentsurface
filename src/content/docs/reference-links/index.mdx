## Agent-Readable Web

| Reference | What It Covers | Use It For |
|---|---|---|
| [Cloudflare Agent Readiness](https://blog.cloudflare.com/agent-readiness/) | Discoverability, content accessibility, bot access control, and capabilities | Understanding what agent-readiness scanners currently check. |
| [Agent Readiness Checker](https://isitagentready.com/) | Public site scanner for agent readiness | Spot-checking whether a URL exposes agent-readable signals. |
| [Markdown for Agents](https://developers.cloudflare.com/fundamentals/reference/markdown-for-agents/) | `Accept: text/markdown`, `x-markdown-tokens`, `Content-Signal`, and edge HTML-to-Markdown conversion | Serving readable pages to agents without wasting context on HTML. |
| [llms.txt](https://llmstxt.org/) | LLM-readable site indexes | Publishing a concise map of docs, APIs, and key resources. |
| [Content Signals](https://contentsignals.org/) | Search, AI input, and AI training policy signals | Declaring how accessed content may be used. |
| [Web Bot Auth](https://datatracker.ietf.org/doc/draft-meunier-web-bot-auth/) | Signed HTTP requests for bot identity | Letting sites distinguish accountable agents from anonymous automation. |
| [API Catalog RFC 9727](https://www.rfc-editor.org/rfc/rfc9727.html) | `/.well-known/api-catalog` | Advertising API specs, docs, status pages, and related API metadata. |
