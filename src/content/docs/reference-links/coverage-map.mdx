## Summary

Agent Surface is a framework-neutral reference for making software legible to agents and for designing production agent systems. It should explain concepts, boundaries, tradeoffs, failure modes, and evaluation criteria before it points to any specific implementation.

Mastra's Guides sidebar is one useful external checklist because it spans frameworks, multi-agent systems, UI, deployment, tutorials, and migrations. The goal is not to mirror Mastra or document Mastra solutions. The goal is to make sure Agent Surface covers the same topic classes in a portable way.

This map was last checked against the Mastra Guides sidebar on April 26, 2026.

## Coverage Standard

A topic is covered when Agent Surface includes:

- the concept and the problem it solves
- when to use it and when not to use it
- architecture and control-flow tradeoffs
- common failure modes
- testing or evaluation guidance
- relevant standards, protocols, or framework examples when useful

Framework commands, provider dashboards, and exact version-specific setup steps belong in the upstream docs unless Agent Surface needs them to explain an audit or scaffolding decision.

## Maintenance

Use external docs as coverage checklists, not as content models. When a major agent framework adds a new guide category, update Agent Surface only if the category exposes a durable reference topic: a new architecture pattern, runtime boundary, tool surface, auth model, evaluation concern, deployment shape, or migration risk.
