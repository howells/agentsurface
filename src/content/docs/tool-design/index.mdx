## Summary

Tool Design is the behavior contract for model-callable actions across frameworks. Use it when defining how an agent chooses, validates, executes, and interprets a tool, whether the tool is backed by an API, CLI, database, MCP server, or local function.

- **Owns:** names, descriptions, input/output schemas, safety annotations, token budget, curation, and portability.
- **Does not own:** HTTP/OpenAPI contracts; see [API Surface](/docs/api-surface). MCP server lifecycle and transports; see [MCP Servers](/docs/mcp-servers).
- **Agent-ready means:** the model can pick the right tool, pass valid parameters, understand the result, and recover from failure without hidden product knowledge.
