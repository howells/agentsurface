## The Problem

Traditional software is built for humans. APIs have documentation sites with marketing copy. CLIs have `--help` text designed for human readers. Error messages assume a person will read them and figure out what to do.

AI agents are different. They cannot ask clarifying questions. They cannot infer context from visual cues. They cannot "just figure it out." Everything that made APIs good for developers — specificity, consistent semantics, helpful errors — matters *exponentially more* when the consumer cannot compensate for ambiguity.

## The Solution

Agentify provides a systematic framework for evaluating and transforming any codebase across eleven dimensions of agent-readiness:

1. **API Surface** — OpenAPI specs, tool definitions, schema design
2. **CLI Design** — Structured output, schema introspection, input hardening
3. **MCP Servers** — Model Context Protocol server implementation
4. **Discovery & AEO** — llms.txt, AGENTS.md, structured data, content negotiation
5. **Authentication** — OAuth 2.1, API keys, agent identity
6. **Error Handling** — Structured errors, recovery hints, retry guidance
7. **Tool Design** — Naming, descriptions, schemas, idempotency, cross-framework portability
8. **Context Files** — AGENTS.md, CLAUDE.md, cursor rules, permission boundaries
9. **Data Retrievability** — Vector embeddings, hybrid search, reranking, knowledge graphs, agentic RAG
10. **Multi-Agent & Orchestration** — Agent delegation, supervisor patterns, state coordination
11. **Testing & Evaluation** — Agent evaluation, tool routing accuracy, pass@k metrics
