## Summary

Error Handling covers structured failures that tell agents what happened and what to do next. Use it for HTTP errors, tool errors, CLI errors, retry policy, validation feedback, and idempotent recovery.

- **Owns:** RFC 9457 Problem Details, agent extensions, retry signals, idempotency keys, recovery suggestions, trace IDs, and CLI error contracts.
- **Does not own:** authentication flows themselves; see [Authentication](/docs/authentication). Tool naming and schemas; see [Tool Design](/docs/tool-design).
- **Agent-ready means:** the failure body is structured enough for the agent to retry, change input, switch tools, wait, or escalate without parsing prose.
