## Summary

Authentication covers how agents receive credentials, prove identity, narrow authority, and call protected APIs or MCP servers without borrowing a human browser session.

- **Owns:** API keys, OAuth for agents, token exchange, DPoP, agent identity, MCP auth metadata, `auth.md`, and replay protection.
- **Does not own:** tool safety semantics; see [Tool Design](/docs/tool-design). Public discovery broadly; see [Discovery](/docs/discovery).
- **Agent-ready means:** credentials are scoped, short-lived where possible, machine-usable at runtime, validated on every request, and auditable by agent/user/delegation chain.

Registration can still involve human consent or an OTP claim ceremony. The runtime request path should stay programmatic: no CAPTCHA, login page, or ambient session cookie dependency.

## Choose A Path

- [API Keys](/docs/authentication/api-keys) — only when keys are scoped, rotatable, and not exposed to models.
- [OAuth 2.1 for Agents](/docs/authentication/oauth-for-agents) — default for standalone agent services and machine-to-machine access.
- [Agent Identity](/docs/authentication/agent-identity) — model agents as first-class principals for audit trails.
- [Token Exchange](/docs/authentication/token-exchange) — narrow authority for delegation and sub-agents.
- [DPoP](/docs/authentication/dpop) — bind tokens to client keys to reduce replay risk.
- [Protected Resource Metadata](/docs/authentication/protected-resource-metadata) — publish OAuth metadata for protected APIs.
- [MCP Authorization](/docs/authentication/mcp-auth-model) — apply OAuth discovery and client identity to MCP.
- [auth.md](/docs/authentication/auth-md) — describe agentic registration and credential-claim flows.
- [Idempotency and Replay Protection](/docs/authentication/idempotency-and-replay) — defend write operations from duplicated or replayed requests.
- [Auth Anti-Patterns](/docs/authentication/anti-patterns) — avoid browser sessions, unscoped tokens, and hidden delegation.
