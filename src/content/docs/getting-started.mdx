## Summary

Agent Surface is organized as a dense guide, not a linear tutorial. Start by choosing the job you have: learn the concepts, build an agent system, expose an existing product to agents, evaluate readiness, or track the standards and tooling landscape.

- New to the topic: start with fundamentals.
- Building agents: start with runtime boundaries and agent scaffolding.
- Making software usable by agents: start with surfaces.
- Comparing maturity: use scoring after you understand the surface.
- Tracking the ecosystem: use protocols, references, and the catalog.

## The Core Map

Agent Surface has five main subject areas:

1. **Agent fundamentals:** design principles, runtime boundaries, frameworks, UI, and protocols.
2. **Agent systems:** agents, workflows, multi-agent patterns, retrieval, memory, testing, and production patterns.
3. **Agent-readable surfaces:** APIs, CLIs, MCP servers, discovery, context files, auth, errors, and tool definitions.
4. **Evaluation:** scorecards, rubrics, evidence, deltas, clustering, and calibration.
5. **Reference:** standards, source links, and tooling.

## If You Are Starting From Zero

Use this implementation order for an existing product:

1. Write accurate context files: `AGENTS.md`, tool-specific overlays, and repository boundaries.
2. Make docs discoverable: `llms.txt`, structured data, sitemap, and stable Markdown routes.
3. Make command output parseable: JSON, NDJSON, stable exit codes, and strict validation.
4. Publish machine-readable contracts: OpenAPI, schema metadata, and tool definitions.
5. Add recovery paths: RFC 9457 errors, retry guidance, idempotency, and trace IDs.
6. Secure programmatic access: scoped credentials, OAuth metadata, DPoP where needed.
7. Expose tools intentionally: MCP server, curated tool sets, resources, prompts, and annotations.
8. Test real workflows: eval cases, traces, red-team suites, CI gates, and score deltas.
