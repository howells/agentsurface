## Retrieval, RAG, and Memory

This section keeps only retrieval tools that directly support RAG, semantic search, reranking, or long-term agent context. Generic databases are intentionally omitted.

| Tool | Website | What it is for | Tier | Notes |
|---|---|---|---|---|
| [QMD](https://github.com/tobi/qmd) | github.com/tobi/qmd | Local-first document and Markdown search with BM25, vectors, reranking, and MCP | Core | Strong fit for local search, extra paths, Markdown memory, and session transcript recall. |
| [Honcho](https://docs.honcho.dev/) | docs.honcho.dev | AI-native memory, cross-session continuity, and user/agent modeling | Core | Relevant when memory should model users, agents, and relationships rather than only retrieved chunks. |
| [Cloudflare Agent Memory](https://blog.cloudflare.com/introducing-agent-memory/) | blog.cloudflare.com | Managed persistent memory for agents with ingestion, recall, explicit remember/forget operations, and exportability | Watch | Private beta, but the shape is important: memory is treated as a constrained agent tool rather than raw database access. |
| [Cloudflare AI Search](https://developers.cloudflare.com/ai-search/) | developers.cloudflare.com | Managed search for applications and agents with automated indexing, hybrid search, MCP endpoints, and UI snippets | Conditional | Good when a docs site, product corpus, or per-tenant file set should become an agent-searchable tool quickly. |

## Document Ingestion, OCR, and Parsing

Use these when agent quality depends on reading messy source material rather than clean Markdown. Document tooling is often the hidden difference between a decent RAG demo and a useful production agent.

| Tool | Website | What it is for | Tier | Notes |
|---|---|---|---|---|
| [LlamaParse](https://docs.cloud.llamaindex.ai/llamaparse) | docs.cloud.llamaindex.ai | Document parsing service from LlamaIndex for PDFs, tables, layouts, and multimodal documents | Core | Strong default for RAG projects already using LlamaIndex or needing high-quality PDF-to-structured-text conversion. |
| [Unstructured](https://docs.unstructured.io/) | docs.unstructured.io | Ingestion and partitioning for PDFs, Office docs, HTML, images, and enterprise files | Core | Good when the ingestion problem spans many file types and connectors. |
| [Docling](https://docling-project.github.io/docling/) | docling-project.github.io | Open-source document conversion, parsing, OCR, layout, table, and multimodal extraction | Conditional | Strong local/open-source path when teams want control over document processing. |
| [Reducto](https://docs.reducto.ai/) | docs.reducto.ai | Document ingestion and extraction API for complex PDFs and structured outputs | Conditional | Worth evaluating for finance, legal, healthcare, and other layout-heavy document workflows. |
| [Mistral OCR](https://docs.mistral.ai/capabilities/document_ai) | docs.mistral.ai | OCR and document AI through Mistral's Document AI stack | Conditional | Good when Mistral is already in the model/provider mix or OCR quality is central. |
| [Azure AI Document Intelligence](https://learn.microsoft.com/azure/ai-services/document-intelligence/) | learn.microsoft.com | Enterprise document extraction for forms, invoices, receipts, layout, and custom models | Conditional | Strong when compliance, Azure integration, and existing enterprise document flows matter. |
| [Google Document AI](https://cloud.google.com/document-ai/docs) | cloud.google.com | Document processors for OCR, forms, invoices, layout, and custom extraction | Conditional | Best when the app and data are already on Google Cloud. |
| [AWS Textract](https://docs.aws.amazon.com/textract/) | docs.aws.amazon.com | AWS document text, table, form, and layout extraction | Conditional | Best when documents already live in S3 or the system is AWS-native. |
| [Chunkr](https://docs.chunkr.ai/) | docs.chunkr.ai | Document parsing API for layout-aware extraction and chunking | Watch | Track for document-heavy RAG where chunk quality is a bottleneck. |
| [Marker](https://github.com/datalab-to/marker) | github.com/datalab-to | Open-source document conversion to Markdown, JSON, and HTML | Conditional | Useful local option for PDF-to-Markdown pipelines and batch ingestion. |
| [MinerU](https://github.com/opendatalab/MinerU) | github.com/opendatalab | Open-source document extraction and conversion toolkit | Watch | Track for open document parsing, especially PDFs with tables, formulas, and layout needs. |
