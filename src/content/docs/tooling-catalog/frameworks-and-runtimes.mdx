## Agent Frameworks and Runtimes

Use these when the product needs agents, tools, memory, workflows, or structured orchestration.

| [OpenHands SDK](https://docs.openhands.dev/sdk/index)                            | docs.openhands.dev        | Software-development agent SDK with Python and REST APIs                                               | Watch       | Interesting for coding-agent products, but keep separate from general product agents.                                         |

## Agent Workflow and Context Tools

These are not always full agent frameworks. They are worth tracking because they give agents safer orchestration, repeatable workflows, context compression, or bounded LLM substeps.

| Tool                                    | Website          | What it is for                                                                                     | Tier        | Notes                                                                                              |
| --------------------------------------- | ---------------- | -------------------------------------------------------------------------------------------------- | ----------- | -------------------------------------------------------------------------------------------------- |
| [OpenProse](https://www.prose.md)       | prose.md         | Markdown-first multi-agent workflow programs with explicit parallelism and reusable `.prose` files | Watch       | Promising for portable, reviewable agent workflow recipes.                                         |
| [Trigger.dev](https://trigger.dev/)     | trigger.dev      | TypeScript workflows, background tasks, retries, checkpointing, and AI agent jobs                  | Conditional | Strong when agent work needs durable execution rather than an in-request loop.                     |
| [Temporal](https://docs.temporal.io/)   | docs.temporal.io | Durable workflow engine with retries, timers, signals, queries, and long-running execution         | Conditional | Use when agent work must survive process restarts, human delays, retries, and multi-day workflows. |
| [Inngest](https://www.inngest.com/docs) | inngest.com      | Durable serverless functions, background jobs, retries, and event-driven workflows                 | Conditional | Good TypeScript path for event-driven agent tasks that should not run inside a request lifecycle.  |
| [Hatchet](https://docs.hatchet.run/)    | docs.hatchet.run | Open-source task queue and durable workflow engine                                                 | Watch       | Useful when teams want self-hostable workflows with retries, concurrency, and observability.       |
| [Prefect](https://docs.prefect.io/)     | docs.prefect.io  | Python workflow orchestration for data and AI pipelines                                            | Conditional | Relevant when agent workflows are coupled to Python data processing or scheduled RAG pipelines.    |
