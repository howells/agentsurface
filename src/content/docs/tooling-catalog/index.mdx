## High-Signal Default Stack

If you only need a practical shortlist, start here. These are not universal defaults; they are the tools that repeatedly show up as useful building blocks for production agent workflows.

| Job | Strong starting points | Why |
|---|---|---|
| Agent runtime | Mastra, Vercel AI SDK, OpenAI Agents SDK, LangGraph | Covers TypeScript apps, OpenAI-native agents, and explicit graph workflows. |
| Tool interoperability | MCP, JSON Schema, OpenAPI, Arazzo | Keeps tools inspectable, portable, and less prompt-dependent. |
| Web search and extraction | Firecrawl, Tavily, Exa, Browserbase, Stagehand, Apify | Covers search, crawling, extraction, JavaScript-heavy browsing, and repeatable browser automation. |
| RAG and retrieval | pgvector, Qdrant, Pinecone, Weaviate, Cloudflare AI Search, Voyage AI, Cohere, Jina AI | Covers vector storage, hybrid search, embeddings, and reranking. |
| Document ingestion | LlamaParse, Unstructured, Docling, Reducto, Mistral OCR, Azure AI Document Intelligence | Turns messy PDFs, docs, tables, and scanned content into retrievable structure. |
| Memory | Honcho, Zep, Letta, Mem0, Supermemory, Graphiti | Covers explicit agent memory, temporal graphs, user memory, and recall layers. |
| External app actions | Composio, Arcade, Pipedream, Zapier, n8n, Paragon | Gives agents controlled access to SaaS actions without writing every connector from scratch. |
| Sandboxed execution | E2B, Daytona, Vercel Sandbox, Cloudflare Sandbox, Modal | Lets agents run code or generated artifacts away from production systems. |
| Durable workflows | Temporal, Trigger.dev, Inngest, Hatchet, LangGraph, Mastra workflows | Keeps multi-step agent work resumable, observable, and retryable. |
| Observability and evals | Braintrust, Langfuse, Arize Phoenix, LangSmith, Promptfoo, RAGAS, DeepEval, OpenTelemetry GenAI | Gives traces, datasets, evaluations, and regression gates. |
| Guardrails and security | Guardrails AI, NeMo Guardrails, Lakera, Llama Guard, Presidio, Pangea | Helps with validation, policy checks, PII handling, prompt-injection risk, and auditability. |

## Watch, Do Not Default

These are worth knowing about but should not become defaults without a concrete reason.

| Area | Examples | Why not default |
|---|---|---|
| Long-tail model hosts | AIHubMix, Chutes, FastRouter, Kilo Gateway, MiniMax, Moonshot/Kimi, Nebius, Requesty, Scaleway, Synthetic, Venice, Volcengine, Z.AI, Zhipu | Provider lists churn quickly. Keep them behind OpenRouter, AI Gateway, LiteLLM, or another router. |
| MCP directories | Smithery, Glama, mcp.so, assorted MCP finder sites | Useful for discovery, but server quality, security, and maintenance vary widely. Prefer official registries and vendor-maintained servers. |
| Tool hubs | Composio, Metorial, Arcade-style tool layers | Useful for breadth, but can hide product semantics and permission boundaries. Evaluate before adopting. |
| Novel agent frameworks | Small orchestration frameworks without clear production adoption | Track ideas, but build foundations around Mastra, AI SDK, OpenAI Agents SDK, LangGraph, MCP, or plain code. |
| Generic SaaS integrations | GitHub, Notion, Linear, Slack, Figma, Stripe, HubSpot, Salesforce | They can be agent tools, but the AI-specific question is how they are exposed: MCP server, OpenAPI tool, official agent connector, or bespoke integration. |
| Messaging channel plugins | DingTalk, QQbot, WeCom, Matrix, Zalo, Microsoft Teams | These are agent surfaces, but the tooling lesson is the plugin/channel pattern. Do not turn the AI tooling catalog into a messaging directory. |
