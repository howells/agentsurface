## Summary

MCP Servers covers protocol-specific implementation for exposing tools, resources, and prompts through the Model Context Protocol. Use it when you are building, hosting, testing, authenticating, or distributing an MCP endpoint.

- **Owns:** MCP architecture, transports, lifecycle, tool/resource/prompt exposure, annotations, auth integration, testing, Next.js hosting, generation, and examples.
- **Does not own:** general tool definition quality; see [Tool Design](/docs/tool-design). HTTP/OpenAPI design before wrapping it in MCP; see [API Surface](/docs/api-surface).
- **Agent-ready means:** clients can connect, inspect capabilities, call tools with schemas, receive structured results/errors, and enforce auth and safety at the server boundary.
