## Summary

CLI Design covers command-line surfaces that agents can script without scraping terminal prose. Use it when a product exposes capabilities through commands, local automation, package scripts, or developer tools.

- **Owns:** command structure, JSON output, raw payload input, schema introspection, exit codes, input hardening, safety flags, and packaging CLI knowledge for agents.
- **Does not own:** model-callable tool schemas in general; see [Tool Design](/docs/tool-design). HTTP service contracts; see [API Surface](/docs/api-surface).
- **Agent-ready means:** every command can be discovered, invoked non-interactively, parsed structurally, retried safely, and diagnosed from exit status plus error body.
